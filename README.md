# Noji Land Builder Dashboard

게임형 토지 활용 현황판 MVP.

## 목표

- 실제 토지 개발/활용 진행상황을 게임처럼 공유
- Land level, zones, objects, actions, quests, players를 한 화면에서 확인
- 가족/친구/동료에게 진행상황과 필요한 도움을 쉽게 설명

## 현재 방식

Vite 기반 정적 웹앱입니다.

- `index.html`: 화면 shell
- `src/app.js`: 렌더링 로직
- `src/styles.css`: 게임 UI 스타일
- `public/data/project.json`: MVP 데이터
- 자체 square grid 좌표 계산
- SVG rect: 1m×1m 정사각형 맵 렌더링

### 맵 조작

- 드래그: 맵 이동 (팬)
- 마우스휠: 커서 위치 기준 확대/축소
- 타일 호버: 구역 정보 + 설명 툴팁
- 레전드 칩 클릭: 해당 구역만 하이라이트
- 우측 하단 HUD: 줌 ＋/−/100%/화면맞춤(⛶)
- 렌더링: 타일/경계선/측정 레이어 분리, 페인트 시 변경 타일만 부분 업데이트

### 인게임 에디터

- 좌측 툴 레일: 보기(V) / 칠하기(B) / 지형(T) / 설치(O) / 측정(M) / ⚙설정
- 컨텍스트 패널: 칠하기 → 구역 팔레트 + 브러시, 지형 → 모드 + 브러시, 설치 → 오브젝트 팔레트 + 방향
- 측정 결과는 맵 위 선 중앙에 m 단위로 직접 표시
- ⚙설정 팝오버: 맵 크기, 파일(저장/불러오기/Export)
- Esc: 설정 닫기 → 측정 취소 → 선택 해제 → 에디터 종료 순

### 지형 (높이/등고선)

- 타일별 정수 고도 `h` (m 단위, −10 ~ +30, 기본 0) — 경사 없음, 심시티식 계단 지형
- 지형 도구 모드: ▲ 클릭마다 +1m / ▼ 클릭마다 −1m / ▦ 지정 높이로 평탄화
- 높을수록 타일이 밝아지는 고도 음영
- 등고선: 고도가 다른 타일 경계에 자동 표시, 5m 배수를 지나는 단차는 굵은 선
- HUD ⛰ 버튼으로 등고선 표시/숨김 (설정 유지)
- HUD ▣ 버튼으로 구역 경계선 표시/숨김 (기본 숨김, 설정 유지)
- 고도는 저장/Export JSON의 타일 `h` 필드에 포함

### 오브젝트 설치

- 시범 오브젝트: 텐트(10×5m), 차량(5×2m), 간이화장실(2×2m), 화로대(1×1m)
- 설치 도구(O): 팔레트에서 선택 → 호버 고스트(초록=가능/빨강=겹침·범위밖·보유없음) → 클릭 설치
- R 또는 ↻ 버튼: 90° 단위 회전 (footprint도 같이 회전, 방향 화살표 표시)
- 기존 오브젝트 클릭(설치 모드) = 집어서 이동 (인벤토리 반환)
- **설치는 인벤토리 보유 수량 안에서만 가능** — 설치 시 1개 차감, 회수 시 반환
- 저장/Export JSON의 `objects` 배열에 포함 (`{id, type, q, r, rot}`)

## 게임 시스템

퀘스트 → 보상 아이템 → 인벤토리 → 설치/레벨업 재료 → 마을 레벨업 → 신규 해금의 루프.

### 퀘스트 보드 (우측 패널)

- 탭: 가능 / 진행 / 완료
- 가능 퀘스트 클릭 → 나에게 할당 (진행으로 이동)
- 진행 퀘스트 "완료하기" 두 번 클릭(실수 방지) → 완료 + 보상 아이템 인벤토리 지급
- 노출 조건: 해금 마을 레벨 + 선행 퀘스트 완료

### 인벤토리 / 인스턴스 (하단 바)

- 아이템별 보유 수량 배지 + 제작 인스턴스 수 + 설치 수 표시, 해금 전 아이템은 🔒
- 카드 클릭 → 해당 아이템의 인스턴스 UI 표시
- **제작** 버튼 → 아이템 템플릿에서 인스턴스 생성
- 인스턴스별 라벨, 소유자, 일반 파일 첨부, 이미지 여러 장 첨부 가능
- 첨부는 정적 앱 MVP 기준으로 브라우저 localStorage에 데이터URL로 저장됨(큰 파일은 서버 저장소 필요)
- **설치 모드** 버튼 → 기존 지도 오브젝트 설치 플로우로 이동

### 퀘스트 재료 / 보상

- 재료: 퀘스트 시작 시 1회 인벤토리에 지급
- 보상: 퀘스트 완료 시 인벤토리에 지급
- 관리 모달에서 재료와 보상을 각각 편집 가능

### 마을 레벨 (좌측 스탯 창)

- 다음 레벨 업그레이드에 필요한 아이템 체크리스트 (보유+설치 합산)
- 모두 수집하면 업그레이드 버튼 활성화 → 레벨업 (재료는 소모되지 않음)
- 레벨업 시 잠겨 있던 지형/아이템/퀘스트가 해금
- 레벨 목록에서 각 레벨에 해금되는 아이템을 역으로 확인 가능

### ⚙ 관리 모달 (헤더 버튼)

- **지형**: 구역 이름·색상·해금 레벨 편집, 추가/삭제 (삭제 시 타일은 Wild로 복귀). 색상 지정 시 테마 무관 고정색, "테마색" 버튼으로 복귀
- **아이템**: 이름·아이콘(이모지)·크기·해금 레벨 편집, 보유 수량 직접 조정(±), 추가/삭제
- **퀘스트**: 퀘스트명·설명·수행 방법·해금 레벨·선행 퀘스트·보상 아이템(복수×수량) 정의, 추가/삭제
- **레벨 조건**: 레벨별 업그레이드 필요 아이템 목록 편집
- 삭제 버튼은 한 번 더 클릭해야 실행 (확인?)

### 데이터

- 게임 상태(구역/아이템/퀘스트/레벨 조건/인벤토리/마을 레벨): `localStorage noji-game-v1`
- 맵(타일/고도/설치물): `localStorage noji-square-map-v1`
- Export JSON 한 파일에 맵+게임 상태 모두 포함, 불러오기로 복원
- `project.json`은 최초 시드(레벨 정의, 초기 퀘스트)로만 사용

## 배포 추천

초기 MVP는 GitHub + Vercel만으로 충분합니다.
Supabase는 다음 단계에서 붙이는 것을 권장합니다.

Supabase가 필요한 시점:

- 여러 사람이 직접 퀘스트 상태를 수정
- 로그인/권한 필요
- 오브젝트 예약/사용 기간 관리
- 코멘트, 사진 업로드, 활동 로그 필요

## 주의

공유용 웹에는 주민번호, 원본 등기 문서, 세금 상세, 가족 개인정보를 올리지 않습니다.
공개 가능한 요약 데이터만 `project.json`에 넣습니다.


## Map Editor

`/map-editor.html`에서 별도 편집기를 제공합니다.

- 1칸 = 1m×1m = 1㎡
- zone paint
- 타일 중심점 기준 m 단위 거리 측정
- 지정된 타일 개수로 ㎡와 평수 자동 계산
- JSON export 지원
